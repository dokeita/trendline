# Steering ルールファイルの使い方

## Steering とは

`.kiro/steering/` に配置するマークダウンファイルで、Kiro への追加の指示やルールを定義できます。プロジェクト固有の規約やワークフローを記述しておくことで、毎回説明しなくても一貫した対応をしてもらえます。

## ファイルの場所

```
.kiro/steering/
├── issues.md         # issue ファイルの作成ルール
└── resolve-issue.md  # issue の実装ルール
```

## 読み込みモード

ファイル先頭の front-matter で読み込み方式を指定します。

| モード | front-matter | 動作 |
|--------|-------------|------|
| 常時読み込み | （なし、またはデフォルト） | すべての会話で自動的に適用される |
| 手動読み込み | `inclusion: manual` | チャットで `#ファイル名` を入力したときだけ適用される |
| ファイルマッチ | `inclusion: fileMatch` + `fileMatchPattern: '*.py'` | 指定パターンのファイルがコンテキストに含まれるとき適用される |

## ルールファイル一覧

### `#issues` — Issue の作成

Issue を追加したいときに使います。

```
#issues issuesに追加して
```

**動作:**
- `issues/` ディレクトリの既存ファイルから次の連番を決定
- `[3桁連番]_yyyyMMdd_タイトル.md` の命名規則でファイルを作成
- テンプレート（概要・現状・対応内容・期待効果・備考）に従って記述

### `#resolve-issue` — Issue の実装

特定の issue を実装したいときに使います。

```
#resolve-issue issue 001 を対応して
```

**動作:**
- 指定番号の issue ファイルを読み込み
- 「対応内容」セクションのタスクをすべて実装
- 実装後にチェックリストを `[x]` に更新

## 新しいルールファイルの作り方

1. `.kiro/steering/` にマークダウンファイルを作成する
2. 先頭に front-matter を記述する（手動読み込みの場合）:

```yaml
---
inclusion: manual
---
```

3. ルールの内容を記述する（概要、手順、テンプレートなど）
4. チャットで `#ファイル名`（拡張子なし）を入力して呼び出す

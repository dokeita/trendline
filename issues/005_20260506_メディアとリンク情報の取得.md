# メディアとリンク情報の取得

## 概要

`expansions` に `attachments.media_keys` を追加し、画像・動画の有無を取得する。また、ポスト内の URL からドメイン情報を抽出し、ニュース記事等への言及を識別できるようにする。

## 現状

- メディア添付の有無が分からない
- リンク先の情報が要約に含まれないため、「何についての話題か」が不明確な場合がある

## 対応内容

- [ ] `fetch_x/index.py`: `expansions=["attachments.media_keys"]`, `media_fields=["type", "url"]` を追加
- [ ] `fetch_x/index.py`: `tweet_fields` に `entities` を追加し、URL のドメインとタイトル（`expanded_url`, `title`）を保存
- [ ] `fetch_x/index.py`: 保存データに `media_type`（image/video/gif）と `urls`（ドメイン・タイトル一覧）を含める
- [ ] `summarize/index.py`: リンク先ドメインやメディア有無をプロンプトに含め、ニュース言及を識別しやすくする

## 期待効果

- ニュース記事への言及が明確になり、要約の情報源が分かりやすくなる
- 画像・動画付き投稿を区別することで、ビジュアルコンテンツのトレンドも把握できる

## 備考

- API コール量が増加する可能性があるため、レート制限に注意
- 費用対効果を見て段階的に導入を検討

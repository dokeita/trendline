# Issues 管理ルール

## 概要

`issues/` ディレクトリにマークダウン形式で対応項目を管理する。

## ファイル命名規則

```
[3桁0埋め連番]_yyyyMMdd_タイトル.md
```

- 連番: `issues/` 内の既存ファイルの最大番号 + 1 から採番する
- 日付: 作成日（JST）を `yyyyMMdd` 形式で記載する
- タイトル: 内容を端的に表す日本語タイトル（スペースはアンダースコアに置換）

例: `001_20260506_public_metricsの保存と要約への反映.md`

## ファイルテンプレート

```markdown
# {タイトル}

## 概要

{この対応項目が何を実現するかを1〜2文で記述}

## 現状

- {現在の問題点や不足している点を箇条書きで記述}

## 対応内容

- [ ] {具体的な作業項目をチェックリスト形式で記述}
- [ ] {対象ファイルパスを明記する}

## 期待効果

- {この対応を行うことで得られるメリットを箇条書きで記述}

## 備考

{依存関係、注意事項、参考情報があれば記載。なければセクションごと省略可}
```

## 作成手順

1. `issues/` ディレクトリ内の既存ファイルを確認し、次の連番を決定する
2. 会話の文脈から対応項目を抽出する
3. 上記テンプレートに従い、各項目ごとに1ファイルを作成する
4. 複数項目がある場合は優先度順に連番を振る（番号が小さいほど優先度が高い）

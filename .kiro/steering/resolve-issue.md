# Issue 対応ルール

## 概要

`issue {番号}` を対応して、と指示されたら、該当する issue ファイルの内容を実装する。

## 手順

1. `issues/` ディレクトリから、指定された番号（3桁0埋め）で始まるファイルを探す
2. そのファイルの「対応内容」セクションに記載されたタスクをすべて実装する
3. 実装後、ビルドやリントが通ることを確認する
4. issue ファイルの対応内容チェックリストを `[x]` に更新する

## 実装時のルール

- 既存コードのスタイル・規約に合わせる
- 対応内容に記載されたファイルパスを正確に参照し、変更対象を特定する
- 変更は最小限にとどめ、issue の範囲外の修正は行わない
- 依存パッケージの追加が必要な場合は明示する

## コミットルール

- 実装完了後、自動ではコミットしない（ユーザーがレビューしてからコミットを指示する）
- コミットを指示された場合のメッセージフォーマット: `{issue番号} {対応内容の要約}`
- 例: `001 public_metricsの保存と要約プロンプトへの反映`
- コミット時に issue ファイルを `issues/closed/` ディレクトリに移動する

## 完了条件

- 「対応内容」のすべてのチェック項目が実装されている
- issue ファイルのチェックリストが `[x]` に更新されている
- コードがエラーなく動作する状態である
- ユーザーにレビューを促し、コミットの指示を待つ
